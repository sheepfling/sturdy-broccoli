# Pitch 202X Java surface probe

Date: `2026-06-23`

Scope:
- Vendor: `Pitch pRTI Free 5.5.10`
- Runtime path: `third_party/pitch/PITCH-prti1516e-manual`
- Bridges exercised:
  - `jpype`
  - `py4j`
- Question answered:
  - whether the bundled Pitch jars expose a live `hla.rti1516_202X` RTI factory and ambassador surface

## Summary

The bundled Pitch runtime does expose a real `202X` Java API surface.

Observed through the repo intake path:

- `RtiFactoryFactory` resolved from `hla.rti1516_202X`
- factory selection succeeded for `Federate Protocol`
- `factory.rtiVersion()` returned `1.0`
- `java_rti.getHLAversion()` returned `IEEE 1516-202X`
- `getRtiAmbassador()` succeeded on both `jpype` and `py4j`
- the resulting ambassador class was `se.pitch.oss.fedpro.client.hla.RTIambassadorClientAdapter`

This does **not** prove IEEE `2025` conformance.

It proves:

- the vendor bundle is not limited to `hla.rti1516e`
- the `202X` lane is not just inert interface ballast
- the current repo can now discover that lane explicitly through `./tools/shim-routes java discover --edition 202X ...`

## Probe command shape

The successful discovery path used the repo intake tool with the explicit jar list
under `third_party/pitch/PITCH-prti1516e-manual/lib`, factory name
`Federate Protocol`, and both bridges:

- `python3 scripts/shim_routes.py java discover --edition 202X --bridge jpype ...`
- `python3 scripts/shim_routes.py java discover --edition 202X --bridge py4j ...`

The reports were emitted to a temporary output directory rather than checked in
directly. The checked-in reusable structural comparison is:

- [pitch_202x_surface_audit_2026-06-23.md](pitch_202x_surface_audit_2026-06-23.md)

## Operational implication

Do not say "Pitch has no post-2010 Java surface" for the current bundled
runtime.

The accurate statement is:

- Pitch does not expose `hla.rti1516_2025`
- Pitch does expose a working vendor-specific `hla.rti1516_202X`
- any route claiming actual IEEE `2025` support still requires an adapter and
  evidence, not just jar discovery
