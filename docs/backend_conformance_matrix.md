## Scope

- Backend columns refer to what a Python federate can rely on through
  `create_rti_ambassador(...)`.
- `certi-jpype` and `certi-py4j` inherit the same runtime status as `certi`;
  separate adapter parity is covered by the Java-profile tests.
- Pitch remains blocked until a locally activated runtime is available.

Use this file to answer:

- which IEEE 1516.1-2010 clauses have working behavior now
- where Python RTI is ahead of vendor/runtime paths
- where CERTI or Pitch remains partial or blocked

Legend:

- `yes`: implemented and exercised by automated tests
- `partial`: implemented, but with narrower semantics or only adapter/unit coverage
- `probe-fails`: service wiring exists, but the local real CERTI runtime fails when exercised
- `blocked`: runtime not currently executable in this workspace

## Notes

- The pure Python RTI remains the easiest place to evolve spec-facing semantics and now covers the core negotiated-ownership response services as well as the acquisition/request/cancel flows.
- The pure Python RTI now models a more CERTI-like ownership state machine for negotiated divestiture, confirm-divestiture transfer, acquisition requests, candidate queues, release denial, cancellation, and release-response transfer.
- The current CERTI runtime in this workspace is strong on exchange, time, synchronization, unconditional divestiture, acquisition-if-available, and ownership query.
- The negotiated ownership path is now wired through the CERTI helper and adapters, and the local runtime is stable enough to distinguish the direct owner release-request branches.
- A patched local CERTI build overlay now exists for `confirmDivestiture`, `attributeOwnershipReleaseDenied`, and the earlier assumption-broadcast crash fix.
- The direct CERTI-side 1516-2010 ownership harness no longer shows the earlier create/join or read-error collapse for these ownership probes; the remaining question is clause-level parity, especially where CERTI currently maps `confirmDivestiture` and `attributeOwnershipDivestitureIfWanted` onto the same runtime path.
- See [certi_negotiated_ownership_findings.md](/Users/rick/Library/Mobile%20Documents/com~apple~CloudDocs/GIT/hla-2010/docs/certi_negotiated_ownership_findings.md) for the source-level investigation and the rationale for treating the current CERTI behavior as a vendor/runtime limitation.
- Add Pitch rows only after activation allows the same real-runtime probes to run.
